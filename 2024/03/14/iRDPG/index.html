iRDPG 論文閱讀

歡迎來到我的部落格！本文是我在深入研究“Adaptive Quantitative Trading: An Imitative Deep Reinforcement Learning Approach”這篇論文後的一些思考和紀錄。請注意，這些內容是基於我的理解和解釋，可能不完全精確。希望大家在閱讀後，也去探索原始論文或資料來形成自己的見解。

論文[1]中提到了在金融市場中要使用AI去做交易或預測目前會遇到兩個問題。

  1. 金融數據具有高噪音、不平穩的問題
  2. 由於存在市場摩擦，agent 需要平衡 exploration 和 exploitation

第一點提到資料不穩定的問題，作者們提出了使用POMDP這個方法迴避掉直接觀察金融數據。

第二點提到需要處理 exploration 和 exploitation 的問題。這是在講說探索(exploration)需要用有系統的方式去探索，如果只使用隨機探索，因為市場的複雜性會造成學習極度緩慢。在使用(exploitation)上也需要一位專家，模型在專家身邊邊看邊學，這樣學習的速度比較快。

POMDP

全名: Partially Observable Markov Decision Process (部分可觀察馬可夫決策過程)

是由馬可夫決策過程衍伸出來的小改版。

先來介紹馬可夫決策(MDP)，MDP是由四個部分組成 (S,A,Pa,Ra)(S, A, P_a, R_a)(S,A,Pa​,Ra​)

  1. SSS表示狀態
  2. AAA表示動作
  3. PaP_aPa​是機率，表示在一個狀態下因為一個特定的動作而轉移到特定的新狀態的機率。
  4. RaR_aRa​是獎勵，表示在一個狀態下因為一個特定的動作而轉移到特定的新狀態所獲得的獎勵。

那在POMDP中，與MDP不同的是，POMDP被設定成無法觀察到所有的狀態。而是加上OOO這個部份來決定模型可以觀測的資料。

OOO是一個function決定模型可以觀測到的資料。

Imitative RDPG

接下來要介紹作者製作的新模型iRDPG。

iRDPG是作者修改RDPG[2]後的模型。history 的資料使用Gate Recurrent Unit (GRU) [3]去控制。

GRU是一個改良LSTM的模型，用來加快執行速度及減少記憶體的耗用。LSTM的輸入有兩個閥遺忘閥 (forget gate) 與輸入閥 (input gate)，而GRU用一個更新閥 (update gate) 取代。

Imitative Learning

接下來要介紹模仿學習。這是為了加快學習所製作出來的方法，不然要重新學習一切都從隨機開始這樣模型會跑超久還在亂買亂賣。作者分兩個部分去加強模型，分別是Demonstration Buffer和Behavior Cloning。

Demonstration Buffer

模型可以參考稱之為專家的行為(就是可以讀取到別人的範例)，像是這篇論文就參考了 Dual Thrust這個策略做的範例，讓模型可以”參考”由 Dual Thrust這個策略所執行的行動去做pre-train，這樣模型就有執行交易的基礎。

在訓練時，作者使用了prioritized experience replay (PER)[4]這個方法。優先回放 (PER)的想法是打破均勻採樣，讓權重更高的數據(更有價值的數據)可以有更多機會被採用。公式可以寫成P(i)=ptϕ∑ipiϕP(i)=\frac{p_t^\phi}{\sum_{i}p_i^\phi}P(i)=∑i​piϕ​ptϕ​​，其中pip_ipi​是episode iii的機率，ϕ\phiϕ是常數。這是原始公式，作者將其公式修改成公式 (10)。

Behavior Cloning

這部分也是依靠專家，但這次的專家不同，這次是創造一個預言性的交易專家，他總是在最低價格買，在最高價格賣。對於每個 training step，使用 Behavior cloning [5]技術來計算模型的行為與預言專家的行為之間的差距。

比較特別的是，這次的Loss Function是有條件的。這種做法叫Q-Filter [6]，條件是如果專家的動作比模型的動作好才會去記錄這個 loss。1Q(hti,aˉti)>Q(hti,μθhti)1_{Q(h_t^i, \bar{a}_t^i)>Q(h_t^i,\mu^\theta h_t^i)}1Q(hti​,aˉti​)>Q(hti​,μθhti​)​這個意思是如果底下的條件成立這個數字就是 1，不成立就是 0，所以這個式子就是專家動作好就 *1 所以會被記錄下來，其他情況就 *0 所以不會被記錄下來。

實驗結果

實驗數據

使用中國股指期貨中的IF和IC的分鐘級數據，IF所指的是沪深300指數，IC是中证500指數。

訓練集使用2016年1月1號到2018年5月8號的資料，測試集使用2018年5月9號到2019年5月8號的資料。

接下來會有四個指標用來比較模型。

  1. Total return rate: Tr≔(Pend−Pstart)/PstartTr \coloneqq (P_{end} - P_{start}) / P_{start}Tr:=(Pend​−Pstart​)/Pstart​，表示期末資產總值PendP_{end}Pend​和期出資產總值PstartP_{start}Pstart​的差異，與期出資產總值的比例。
  2. Sharpe ratio: Sr≔E[r]/σ[r]Sr \coloneqq \mathbb{E}[r]/\sigma[r]Sr:=E[r]/σ[r] 代表投資者額外承受的每一單位風險所獲得的額外收益。意思就是在相同風險下獲利越多數值就越大。
  3. Volatility: Vol≔σ[r]Vol \coloneqq \sigma[r]Vol:=σ[r] 這個數值是用來衡量回報率的不確定性，數值高代表回報率的波動大，風險越高。
  4. Maximum Drawdown: Mdd≔max(Pi−Pj)/(Pi),j>iMdd \coloneqq max(P_i-P_j)/(P_i), j>iMdd:=max(Pi​−Pj​)/(Pi​),j>i 這是在衡量移動時間內資產價值從峰值下降到谷底的最大比例跌幅。PiP_iPi​ 代表時段iii的資產價值。表示投資組合潛在的最大損失，用來評估策略的風險程度。

Generalization Ability

再來是泛化能力。這是在比較說模型是否可以適用於所有狀況，像是是否適用於其他的商品。作者使用了兩種不同的資料集，就是前面有提到的IF和IC，加上另一個策略去跟iRDPG模型做比較。結果在Table 3 和 Figure 6。

可以看到對於Dual Thrust來說是沒有泛化能力的，因為從 Figure 3 中可以發現IF和IC的趨勢是相同的，但Dual Thrust卻沒有相似的成效，這進一步證明基於靜態技術指標的策略性泛化能力較差。

那看到iRDPG，可以發現雖然IF的結果不如Dual Thrust但是在IC的結果卻是遠遠超過Dual Thrust。而且iRPDG並沒有接受IC的資料集。這樣的表現證明此模型適用於不同的市場，模型可以提取穩健的特徵並學習動態交易策略。
